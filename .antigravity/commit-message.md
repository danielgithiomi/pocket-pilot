# Commit Message Rules

Always generate commit messages using the following format:

`<type>(<scope>): PP-<number> <summary>`

Examples:

- setup(root-config): PP-001 add commitlint and contributing guidelines
- feat(wallet): PP-214 add balance sync endpoint
- fix(http): PP-118 handle timeout retries

Rules:

- Ticket ID is mandatory
- Use lowercase type
- Use imperative mood
- Do not omit the scope
